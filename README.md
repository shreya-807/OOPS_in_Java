# ☕ Java OOP Learning Repository

Welcome to a dedicated space for learning and mastering Object-Oriented Programming (OOP) in Java. 

This repository is designed as a growing, hands-on study guide. Rather than focusing on a single massive application, it contains a collection of simple, modular projects. Each folder is built to demonstrate core programming principles so they can be easily studied, understood, and practiced.

## 🧠 Core OOP Concepts Covered

Across the various examples in this repository, the four main pillars of OOP are actively demonstrated:

* **Encapsulation (Protecting Data):** Keeping sensitive data hidden inside classes and only allowing access through safe, controlled methods (like getters and setters).
* **Inheritance (Reusing Code):** Creating new classes that automatically adopt properties and behaviors from existing base classes, eliminating duplicate code.
* **Abstraction (Hiding Complexity):** Using interfaces and abstract classes to define *what* a system must do, without exposing the complex details of *how* it does it.
* **Polymorphism (Flexibility):** Designing systems that can treat different objects the same way, allowing a single standard command to trigger different, specific behaviors based on the object type.

## 📖 How to Study the Code

For the best learning experience, follow this approach when opening any project folder:
1. **Find the Blueprints:** Look for the core `interfaces` and abstract `classes` first to understand the rules.
2. **See the Implementation:** Check how the specific objects (like the different card types) implement or extend those base rules.
3. **Watch it Run:** Open the main execution file (usually `Client.java` or `Main.java`) to see how the objects interact together.

## 🛠️ How to Run and Test

1. Ensure Java is installed on the system.
2. Clone this repository to a local machine.
3. Open the folder in an IDE (such as VS Code, IntelliJ, or Eclipse).
4. Navigate to the specific project folder to be tested.
5. Compile the `.java` files.
6. Run the main client file to see the code in action!
